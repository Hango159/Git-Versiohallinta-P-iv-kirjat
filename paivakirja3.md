# Oppimispäiväkirja: Git projektissa

**Mitä hyötyä voisi olla versionhallinnasta, jos kehität projektia yksin?**
Se toimii projektin "aikakoneena", jolloin voit helposti palata aiempaan toimivaan versioon, jos teet virheen tai haluat kokeilla uusia ideoita turvallisesti haaroitusten avulla.

**Mitä hyötyä voisi olla versionhallinnasta, jos projektissa on useita kehittäjiä?**
Se mahdollistaa samanaikaisen työskentelyn ilman, että koodit ylikirjoittavat toisiaan, ja tarjoaa Pull Request -työkalut koodin katselmointiin sekä muutosten keskusteluun.

**Miten järjestäisit projektitiimin versionhallinnan 3-4 hengen ohjelmistoprojektikurssilla? Laadi tiimiläisille lyhyt ohje, miten projektissa toimitaan.**
Käytetään keskitettyä main-haaraa vakaalle koodille ja develop-haaraa kehitykselle; jokainen uusi ominaisuus tehdään omassa haarassaan ja yhdistetään Pull Requestin kautta vasta katselmoinnin jälkeen.

**Kommenttini opintojaksosta, esim. sisällöstä, materiaalista, työmäärästä, hyödyllisyydestä, työmäärästä. Mitä toivoisit olevan enemmän, mitä vähemmän?**
Kurssi on ollut erittäin hyödyllinen ja käytännönläheinen, mutta toivoisin enemmän esimerkkejä monimutkaisten merge-konfliktien ratkaisemisesta ja vähemmän toistoa peruskomennoissa.
